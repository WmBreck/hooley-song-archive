# Product Vision

## Primary Purpose

The Hooley the Poet archive exists first to keep one organized, durable place for song lyrics.

The first major output is a songbook containing selected lyrics, song images, and short links to listen to the selected song versions.

## Core Publishing Goal

Create a curated lyric-and-song collection that can support:

- a printed songbook
- an electronic songbook with links
- HooleyThePoet.com
- SoundCloud playlists
- YouTube videos
- TikTok promotion
- custom pitch playlists for musicians, producers, and industry contacts

## Best Of Hooley the Poet

The SoundCloud playlist called "The Best Of Hooley" is an important curated source of truth.

It represents the songs and versions most likely to be shared publicly or included in book and promotional projects.

The archive should preserve the distinction between:

- a song as a creative work
- multiple generated or published versions of that song
- the version selected for the Best Of playlist
- the version selected for a book, website, pitch playlist, or other project

## Duplicate Versions

Many downloaded songs may be alternate versions of the same underlying song.

The archive must support duplicate or alternate versions without confusing them with separate songs.

Play counts and listener response on SoundCloud are useful signals, but not perfect proof of quality. A high-performing version may reflect the song, the cover image, the SoundCloud algorithm, timing, or luck.

## Book Use Case

The songbook should eventually include, for each selected song:

- title
- lyrics
- image or cover art
- shortened listening link
- optional notes or commentary

The electronic version should allow readers to click through to hear the selected recording.

## Pitch Playlist Use Case

The archive should eventually allow quick creation of custom playlists for specific audiences, including musicians, singers, producers, and industry contacts.

For example, a Nashville contact might receive a custom playlist selected by genre, mood, voice, subject matter, or commercial potential.

## Future Intake Goal

When a new song is created, the future app should allow a share link to be pasted into an interface and should attempt to pull metadata automatically.

The user should not have to manually enter the same title, link, image, and platform metadata repeatedly.

## Design Direction

The system should be built around these durable objects:

- songs
- song versions
- projects
- assets
- playlists or collections
- links

The website is one output. The deeper value is the archive itself.
